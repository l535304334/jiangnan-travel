三、AI 智慧核心详细设计

3.1 智能客服引擎

技术方案：用户发送自然语言 → 后端接收 → 意图识别（关键词+LLM判断）→ 调用 DeepSeek API → 返回对话。附带 t_ai_chat_log 记录。

降级方案：DeepSeek 不可用时自动切换为关键词匹配模式，返回固定帮助文档。

System Prompt 要点：预置江西文旅知识库（宁都翠微峰/起义纪念馆、赣州古城墙/八境台、南昌滕王阁/八一起义纪念馆等），用户询问文旅问题时AI能简要回答。

3.2 出行预测引擎

算法：按时段（早6-12/午12-18/晚18-22/夜22-6）统计用户过去30天历史订单中最频繁的目的地 Top 3，新用户（冷启动）返回平台热门目的地 Top 5。直接从 t_order 表 GROUP BY 查询，无需快照表。

补充：常用路线快捷入口 — 统计用户过去30天最高频的起终点对 Top 2，首页一键填入。

3.3 动态调价引擎

算法：供需比 = 当前区域（半径3km内）待接单数 / 在线空闲司机数。系数映射：供需比 ≤1 → 1.0，1-1.5 → 1.0-1.2，1.5-2 → 1.2-1.5，≥2 → 1.5（上限）。

前端展示：同时显示原价和上浮后价格，文案为"当前用车需求较高，价格略有上浮"。

3.4 智能调度引擎（多因素加权评分）

因子权重说明
距离分35%司机距乘客越近分越高（归一化0-100）
服务分25%历史评分均值 × 20（默认80）
匹配度分20%车型一致=100，不一致=0
活跃度分10%鼓励冷门司机接单
随机分5%0-100随机数，避免同一司机一直中选
优选加权5%乘客5星评价过的司机 +5分

分布式锁防重单：使用 Redisson RLock（看门狗自动续期），评分最高司机被锁定15秒后释放。

3.5 安全风控引擎

编号规则触发MVP
R1同设备1小时注册 > 3个账号限制下单可选
R2同用户1小时取消 > 5次当日禁下单必做
R3司机轨迹偏离规划路线 > 3km告警管理员可选
R4深夜(23-5)长途跨城安全提示+建议分享必做
R5同IP大量请求下单接口触发验证码可选
R6司机连续在线 > 8小时休息提醒可选
R7司机行程 > 2小时未到达休息建议+服务区推荐必做
