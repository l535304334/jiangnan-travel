# Documentation Skill

## Overview
为 Spring Boot + Vue 全栈项目生成高质量的技术文档。涵盖 API 文档、数据库设计文档、项目架构文档、部署文档等。

## When to Use
- 需要为 API 接口编写文档时
- 需要更新 README.md 时
- 需要记录数据库设计时
- 需要编写架构设计文档时
- 需要记录部署/运维步骤时

## Documentation Types

### 1. API 接口文档
- 每个 API 包含: URL, Method, Request/Response 格式, 示例
- 遵循 RESTful 规范
- 标注认证方式 (JWT Token)
- 说明参数校验规则
- 记录错误码和异常情况

### 2. 数据库设计文档
- 表名和功能说明
- 字段名、类型、注释
- 索引策略
- 关联关系 (外键)
- 变更记录

### 3. 项目架构文档
- 技术栈说明 (版本号)
- 项目结构目录说明
- 模块职责描述
- 核心流程说明 (认证流程、订单流程等)
- 部署架构图

### 4. README 规范
- 项目简介
- 技术栈
- 快速启动 (环境要求、安装步骤)
- 项目结构
- API 概览
- 测试账号
- 部署说明

### 5. 代码注释规范
- 类注释: 说明类的职责和使用方式
- 方法注释: 说明功能、参数、返回值、异常
- 复杂逻辑: 补充行内注释说明意图
- 配置项: 说明配置的作用和可选值

## Process
1. 确定文档类型和目标读者
2. 收集项目源码信息
3. 分析 API/数据库/架构信息
4. 编写结构化文档
5. 检查准确性和完整性
6. 提交审核
